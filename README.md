# Wanderlust

A full-stack Airbnb-inspired accommodation listing platform where users can create listings, upload property images, discover destinations on an interactive map, leave reviews, and manage their own properties.

Built using Node.js, Express, MongoDB, Passport.js authentication, Cloudinary image storage, and Mapbox geolocation services.

# Authorization Rules

## Listing Ownership

Only listing owners can:

* Edit listings
* Delete listings

Implemented through:

```javascript
isOwner
```

middleware.

---

## Review Ownership

Only review authors can:

* Delete reviews

Implemented through:

```javascript
isReviewAuthor
```

middleware.

---

## Protected Routes

Routes requiring authentication use:

```javascript
isLoggedIn
```

middleware.

Unauthenticated users are redirected to login and returned to their original page after successful authentication.

# Key Design Decisions

## Cloudinary Image Storage

Images are stored externally in Cloudinary instead of MongoDB.

Benefits:

* Reduced database size
* Faster image delivery
* Automatic image transformations
* CDN support

---

## GeoJSON Storage

Listing locations are stored as GeoJSON Points.

```json
{
  "type": "Point",
  "coordinates": [longitude, latitude]
}
```

Benefits:

* Future geospatial querying support
* Native map integration
* Efficient location storage

---

## Controller Separation

Business logic is organized into dedicated controllers:

```text
controllers/
в”њв”Ђв”Ђ listings.js
в”њв”Ђв”Ђ reviews.js
в””в”Ђв”Ђ users.js
```

Each controller manages a single domain.

---

## Middleware-Based Authorization

Authentication and authorization are centralized through middleware rather than repeated inside controllers.

Benefits:

* Cleaner controller code
* Reusable security rules
* Easier maintenance

---

## Automatic Cascade Review Cleanup

When a listing is deleted:

```text
Listing Deleted
      в†“
All Associated Reviews Deleted
```

Implemented using a Mongoose post middleware hook.

This prevents orphaned reviews.

# Error Handling

The application uses:

```javascript
wrapAsync()
```

to capture asynchronous errors and forward them to Express error handlers.

Custom errors are represented through:

```javascript
ExpressError
```

which stores:

* HTTP Status Code
* Error Message

All unhandled routes return:

```text
404 Page Not Found
```
